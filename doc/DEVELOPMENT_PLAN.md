## Технологический стек

### Фронтенд

| Технология | Назначение | Почему выбрана |
|------------|------------|----------------|
| Next.js 14+ (App Router) | Фреймворк | SSR/ISR из коробки, файловый роутинг, API Routes для серверной логики |
| React 18+ | UI-библиотека | Компонентный подход, hooks, экосистема |
| Tailwind CSS | Стилизация | Быстрая вёрстка utility-классами, адаптивность, минимум кастомного CSS |
| Tiptap | WYSIWYG-редактор | Headless-редактор на ProseMirror, гибкий, хранит контент в JSON |

### Бэкенд и данные

| Технология | Назначение | Почему выбрана |
|------------|------------|----------------|
| Supabase | BaaS (Backend as a Service) | PostgreSQL, Auth, Storage, Realtime — всё в одном, бесплатный тариф для старта |
| PostgreSQL | База данных | Через Supabase, полнотекстовый поиск, JSONB для контента |
| Supabase Auth | Аутентификация | Email + пароль, JWT, Row Level Security |
| Supabase Storage | Хранилище файлов | Загрузка изображений, публичные бакеты для обложек |

### Инфраструктура

| Технология | Назначение | Почему выбрана |
|------------|------------|----------------|
| Beget | Хостинг + DNS | Российский хостинг, мигрирован DNS с Reg.ru, поддержка Node.js |
| Yandex Cloud | Git-репозиторий + CI/CD | Managed GitLab или Yandex Cloud Source, данные в РФ |
| Яндекс.Метрика | Аналитика | Стандарт для рунета, бесплатная, карты кликов и вебвизор |

### Инструменты разработки

| Инструмент | Назначение |
|------------|------------|
| Claude Code | Основной инструмент разработки — написание кода, рефакторинг, дебаг |
| VS Code | Редактор (при необходимости ручных правок) |
| Supabase Dashboard | Управление БД, просмотр данных, настройка RLS |
| CapCut | Монтаж контента для Reels (документирование процесса разработки) |

## Цветовая схема (правило 60-30-10)

| Доля | Роль | Цвет | Tailwind | Применение |
|------|------|------|----------|------------|
| **60%** | Нейтральный | Белый / серый | `white`, `gray-50`..`gray-900` | Фон, карточки, текст |
| **30%** | Дополнительный | Синий | `brand-600` (#2563eb), `brand-700` (#1d4ed8) | Ссылки, заголовки, кнопки, навигация |
| **10%** | Акцентный | Красный | `red-600` (#dc2626) | Срочные новости, badge «Срочно», важные метки |

Принцип: тёмный текст на белом фоне для максимальной читаемости. Синий — для интерактивных элементов. Красный — только для акцентов (срочные новости).

## Структура базы данных

Полная SQL-схема описана в README.md. Здесь — краткая сводка таблиц и их связей.

| Таблица | Назначение | Ключевые поля |
|---------|------------|---------------|
| `articles` | Все публикации (новости, статьи, материалы) | title, slug, content (JSONB), category_id, status, is_featured |
| `categories` | Разделы и подкатегории | name, slug, parent_id |
| `tags` | Теги | name, slug, is_hot |
| `article_tags` | Связь статей и тегов (many-to-many) | article_id, tag_id |
| `dictionary` | Словарь ингушского языка | word_ingush, word_russian, transcription, audio_url |
| `users` | Администраторы и редакторы | email, display_name, role (admin / editor) |
| `media` | Загруженные изображения | url, filename, alt_text |

### Связи

```
users ──< articles ──< article_tags >── tags
              │
              └── categories (parent_id → categories)

users ──< media

dictionary (отдельная, без связей)
```

### Row Level Security (RLS)

| Таблица | SELECT | INSERT / UPDATE / DELETE |
|---------|--------|--------------------------|
| articles (published) | Все | Только авторизованные (admin, editor) |
| articles (draft) | Только авторизованные | Только авторизованные |
| categories | Все | Только admin |
| tags | Все | Только авторизованные |
| dictionary | Все | Только авторизованные |
| users | Только свой профиль | Только admin |
| media | Все (публичный бакет) | Только авторизованные |

### Фаза 1 — Фундамент (Неделя 1–2)

**Цель:** рабочий каркас сайта, базовая навигация, отображение контента из Supabase.

#### Задачи

**1.1. Инициализация проекта**
- Создать проект: `npx create-next-app@latest nazranka` (App Router, TypeScript, Tailwind CSS)
- Установить зависимости: `@supabase/supabase-js`, `@supabase/ssr`
- Настроить `.env.local` с ключами Supabase
- Создать `lib/supabase.ts` — клиент для browser и server
- Создать `lib/types.ts` — типы Article, Category, Tag, User

**1.2. Настройка Supabase**
- Создать проект в Supabase Dashboard
- Выполнить SQL-миграцию: таблицы articles, categories, tags, article_tags, users, media
- Настроить базовые RLS-политики (SELECT для всех, INSERT/UPDATE для авторизованных)
- Создать Storage bucket `covers` (публичный) для обложек статей
- Заполнить categories начальными данными: Новости, Культура, История, Ингушский язык

**1.3. Главная страница (`app/page.tsx`)**
- Шапка: логотип nazranka.ru, навигация (Новости, Культура, История, Ингушский язык), иконка поиска
- Мобильное меню (бургер)
- Новостная лента: сетка карточек (обложка, заголовок, лид, дата, категория)
- Боковая колонка: блок «Последние новости» (5 штук, заголовок + дата)
- Подвал: ссылки на разделы, соцсети (Telegram, VK), копирайт WEZARIO

**1.4. Страница статьи (`app/[category]/[slug]/page.tsx`)**
- Полный текст статьи из JSONB-поля content
- Обложка, заголовок, лид, дата публикации, категория
- Боковая колонка: похожие материалы из той же категории
- Хлебные крошки: Главная → Категория → Статья

**1.5. Страница раздела (`app/[category]/page.tsx`)**
- Список статей выбранной категории
- Карточки с пагинацией (10 на страницу)

**1.6. Страница тега (`app/tag/[slug]/page.tsx`)**
- Список статей по тегу

**1.7. Мобильная адаптация**
- Tailwind breakpoints: sm, md, lg
- Hamburger-меню
- Одноколоночная лента на мобильном

**1.8. Деплой**
- Сборка: `npm run build`
- Деплой на Beget (Node.js или статический экспорт)
- Привязка домена nazranka.ru

#### Критерии готовности фазы 1

- [ ] Сайт открывается по адресу nazranka.ru
- [ ] Главная показывает статьи из Supabase
- [ ] Клик по карточке открывает полную статью
- [ ] Навигация по разделам работает
- [ ] Сайт корректно отображается на мобильном
- [ ] Тестовые статьи (3–5 штук) добавлены вручную через Supabase Dashboard

### Фаза 2 — Админ-панель (Неделя 3–4)

**Цель:** полноценная админ-панель для управления всем контентом без кода.

#### Задачи

**2.1. Аутентификация**
- Страница входа: `app/auth/login/page.tsx` (email + пароль)
- Supabase Auth: `signInWithPassword`
- Middleware для защиты роутов `/admin/*` — редирект на `/auth/login` если не авторизован
- Проверка роли (admin / editor) из таблицы users

**2.2. Layout админки (`app/admin/layout.tsx`)**
- Боковое меню: Дашборд, Публикации, Разделы, Теги, Медиатека
- Шапка админки: имя пользователя, кнопка выхода
- Адаптив: сворачиваемый сайдбар на мобильном

**2.3. Список публикаций (`app/admin/articles/page.tsx`)**
- Таблица: заголовок, раздел, статус, дата, действия (редактировать / удалить)
- Поиск по заголовку
- Фильтр по статусу: все / черновик / опубликовано
- Фильтр по разделу: Новости, Культура, История, Ингушский язык
- Кнопка «Новая публикация»

**2.4. Форма публикации (`app/admin/articles/new/page.tsx` и `[id]/edit/page.tsx`)**
- Поле заголовка
- Поле краткого описания (лид)
- WYSIWYG-редактор (Tiptap): текст, заголовки, цитаты, ссылки, вставка изображений
- Загрузка обложки (drag-and-drop → Supabase Storage → URL в cover_image)
- Выпадающий список категории
- Мультиселект тегов (с возможностью создать новый тег)
- SEO-поля: meta title, meta description
- Переключатель: черновик / опубликовано
- Кнопка «Сохранить»
- Автогенерация slug из заголовка (транслитерация)

**2.5. Управление разделами (`app/admin/categories/page.tsx`)**
- Список категорий с drag-and-drop для сортировки
- Создание, переименование, удаление категории
- Поддержка подкатегорий (parent_id)

**2.6. Управление тегами (`app/admin/tags/page.tsx`)**
- Список тегов
- Создание, переименование, удаление
- Переключатель is_hot (показывать в ленте горячих тем)

**2.7. Управление главной**
- Переключатель is_featured на карточке статьи
- Drag-and-drop порядок (sort_order) для закреплённых материалов

#### Критерии готовности фазы 2

- [ ] Вход по email + пароль работает
- [ ] Неавторизованный пользователь не видит `/admin`
- [ ] Можно создать статью с обложкой, текстом, тегами — и она появляется на сайте
- [ ] Можно отредактировать и удалить статью
- [ ] Можно сохранить черновик (не появляется на сайте)
- [ ] Управление разделами и тегами работает
- [ ] Закрепление статьи на главной работает

### Фаза 3 — Контент и виджеты (Неделя 5–6)

**Цель:** живой сайт с реальным контентом и полезными виджетами.

#### Задачи

**3.1. Наполнение контентом**
- 15–25 статей по разделам через админку
- Минимум 5 новостей, 5 статей о культуре, 5 об истории, 3–5 по ингушскому языку
- Подбор обложек

**3.2. Горячие темы**
- Горизонтальная прокручиваемая лента тегов с is_hot=true в шапке сайта
- Клик → страница тега

**3.3. Блок «Популярное»**
- Боковой виджет: топ-5 статей по views за последние 7 дней
- Инкремент views при открытии статьи (server action или API route)

**3.4. Поиск**
- Страница `/search?q=...`
- Supabase Full Text Search по title + lead + content
- Результаты: карточки с подсветкой совпадений

**3.5. Фильтрация**
- На страницах разделов: фильтр по тегам
- Клик по тегу в карточке статьи → страница тега

**3.6. Open Graph и социальные превью**
- `generateMetadata()` в Next.js для каждой статьи
- og:title, og:description, og:image (обложка)
- Корректное превью при отправке ссылки в Telegram / VK

**3.7. Яндекс.Метрика**
- Подключение счётчика через `next/script`
- Настройка целей: просмотр статьи, переход по категории

**3.8. Медиатека в админке**
- Страница `/admin/media`: галерея загруженных изображений
- Загрузка новых (drag-and-drop)
- Вставка из медиатеки в Tiptap-редактор

#### Критерии готовности фазы 3

- [ ] На сайте 15+ реальных статей с обложками
- [ ] Горячие темы отображаются в шапке
- [ ] Поиск находит статьи по ключевым словам
- [ ] Ссылка на статью в Telegram показывает превью с картинкой
- [ ] Яндекс.Метрика собирает данные
- [ ] Медиатека в админке работает

### Фаза 4 — Оптимизация и инфраструктура (Неделя 7–8)

**Цель:** скорость, SEO, стабильность, контроль версий.

#### Задачи

**4.1. Git и Yandex Cloud**
- Инициализация Git-репозитория локально: `git init`
- Создание репозитория в Yandex Cloud (Managed GitLab или Yandex Cloud Source)
- Настройка `.gitignore` (node_modules, .env.local, .next)
- Структура веток: main (продакшен) + dev (разработка)
- Первый коммит всего проекта
- Push в Yandex Cloud

**4.2. CI/CD**
- Настройка пайплайна: push в main → сборка → деплой на Beget
- GitLab CI (`.gitlab-ci.yml`) или аналог в Yandex Cloud

**4.3. SEO**
- `sitemap.xml` — автогенерация из списка статей и категорий
- `robots.txt` — разрешить индексацию, закрыть `/admin`
- Schema.org разметка `NewsArticle` на страницах статей
- `<meta>` теги: title, description из SEO-полей или автогенерация из заголовка и лида

**4.4. Производительность**
- Next.js ISR (Incremental Static Regeneration) для страниц статей: revalidate каждые 60 секунд
- Оптимизация изображений: `next/image` с автоконвертацией в WebP
- Lazy loading для изображений ниже фолда
- Минификация CSS/JS (встроено в Next.js)

**4.5. Пагинация**
- На страницах категорий и тегов: 10 статей на страницу
- URL: `/novosti?page=2`

**4.6. RSS-лента**
- API route `/api/rss` → XML-фид последних 20 статей
- Автообнаружение: `<link rel="alternate" type="application/rss+xml">`

#### Критерии готовности фазы 4

- [ ] Код хранится в Yandex Cloud, push в main деплоит на Beget
- [ ] Структура веток main + dev настроена
- [ ] sitemap.xml и robots.txt доступны
- [ ] Lighthouse Performance score ≥ 80
- [ ] Страницы статей кэшируются (ISR)
- [ ] RSS-лента доступна

## Риски и решения

| Риск | Вероятность | Решение |
|------|-------------|---------|
| Beget не поддерживает SSR Next.js | Средняя | Использовать `output: 'export'` (статический экспорт) или перейти на Vercel/VPS |
| Supabase Free Tier исчерпан | Низкая (на старте) | 500 МБ БД, 1 ГБ Storage — достаточно для первого года. При росте — Pro план ($25/мес) |
| Нет контента при запуске | Высокая | Фаза 3 целиком посвящена наполнению. Минимум 15 статей до публичного запуска |
| Сложность Tiptap | Средняя | Начать с starter-kit, расширения добавлять постепенно |
| DNS-проблемы после миграции | Низкая | DNS уже мигрирован на Beget, пропагация завершена |
